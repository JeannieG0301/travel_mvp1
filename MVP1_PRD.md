# MVP1 产品需求文档

> 本文档为 MVP1 的产品需求权威定义。**功能变更或维护时请及时更新。**
>
> 关联文档：[INPUT_PARAMS_SPEC.md](./INPUT_PARAMS_SPEC.md)（用户输入详细规范）、[OUTPUT_FORMAT_SPEC.md](./OUTPUT_FORMAT_SPEC.md)（行程输出格式规范）

---

## 一、产品概述

- **产品定位**：基于 LLM 的通用旅游规划平台，根据用户输入生成旅行行程
- **MVP1 首版板块**：新西兰自驾
- **目标用户**：中国用户落地新西兰后的行程规划
- **端形态**：Web（详见 [PLATFORM_DECISION.md](./PLATFORM_DECISION.md)）
- **获客渠道**：早期可能来自小红书等，通过链接分享
- **未来方向**：可能面向国际市场，为境外游客入境中国游做行程规划

---

## 二、MVP1 功能范围

| 维度 | 范围 |
|------|------|
| 落地城市 | 仅支持奥克兰、基督城 |
| 游览区域 | 北岛 / 南岛 / 南北岛都玩（南北岛可含国内航班/渡轮建议） |
| 行程输出 | 轻度结构化（title + plan + days + tips） |
| 对话模式 | 单轮：一次输入 → 一次生成 |
| 用户身份 | 匿名使用，不登录，无历史保存 |
| 目的地 | 仅新西兰 |
| 交通方式 | 以自驾为主 |
| 餐饮 | 不包含在输出中 |

---

## 三、核心决策记录

| 决策点 | 结论 |
|--------|------|
| 落地城市 | 保持奥克兰、基督城 |
| 南北岛都玩 | 保留，可含国内航班/渡轮 |
| 行程输出结构 | 轻度结构化（按天 + 上午/下午/晚上） |
| 多轮对话 | MVP1 不做，单轮即可 |
| 登录/账号 | MVP1 不做，匿名使用 |
| tips 内容 | 预定提醒（按淡旺季）、装备建议等；不接入路况/天气 API |
| 交通描述 | 自然语言，含地点→地点及驾车时长 |

---

## 四、用户输入

用户通过表单提交需求，系统据此生成行程。

**必填**：落地城市、游览区域、出发月份、游玩天数、出行人数  
**选填**：落地时间、离程时间、旅行风格、预算级别、必去清单

详见：[INPUT_PARAMS_SPEC.md](./INPUT_PARAMS_SPEC.md)

---

## 五、行程输出

LLM 返回的行程为 JSON，包含：标题、总览、按日行程（上午/下午/晚上）、注意事项。

每个时段含：交通、景点、活动、住宿。交通为自然语言，需包含地点间驾车时长。

详见：[OUTPUT_FORMAT_SPEC.md](./OUTPUT_FORMAT_SPEC.md)

---

## 六、排除范围（MVP1 不做）

- 机票预订 / 航班查询
- 酒店 / 住宿预订
- 租车对接
- 其他目的地板块（日本、澳洲等）
- 用户登录 / 历史行程 / 收藏
- 多轮对话 / 行程微调追问
- 路况、天气 API 接入
- 餐饮推荐
