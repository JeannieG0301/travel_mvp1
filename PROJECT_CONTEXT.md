# Travel Planner 多智能体项目上下文

> 供 AI 助手和团队成员快速了解项目背景。新建对话时建议先阅读此文档。

## 项目概述

AI 旅行规划助手，采用多智能体协作方式开发。开发阶段在 Cursor IDE 中配置了 7 个角色，模拟软件团队协作。

## 七个角色

> **角色定义详见 `ROLES.md`**（职责、产出物、依赖、协作顺序）。

| 角色 | 职责 |
|-----|------|
| 项目经理 | 整体把控、管理进度文档、确定下一任务、为 responsible 角色生成 prompt |
| 产品经理 | 需求定义、用户故事、功能优先级 |
| UI/UX 设计师 | 界面设计、交互流程、视觉规范、原型 |
| 架构师 | 系统设计、技术选型、模块划分 |
| Tech Lead | 技术落地、代码规范、技术难点 |
| 开发团队 | 编码实现、功能开发 |
| 测试团队 | 测试用例、质量验证、回归测试 |

## 当前工作流

```
项目经理 → 更新进度文档 → 确定下一任务 → 生成给 responsible 角色的 prompt
                                                      ↓
                                              角色执行任务
                                                      ↓
                                              生成汇报 prompt
                                                      ↓
                                        手动复制 → 粘贴到项目经理对话框
                                                      ↓
                                              项目经理继续...
```

## 待解决问题

- 减少手动复制粘贴的沟通成本
- 角色变更后，各 Cursor Agent 需同步（见 `docs/AGENT_ROLE_SYNC.md`）
- 探索自动化编排或更顺畅的协作方式（如：Cursor Composer、Rules、工作流脚本、多智能体框架等）

## 相关讨论

- OpenClaw 可作为多通道入口，未来可能将旅行规划助手接入 WhatsApp、Telegram 等
