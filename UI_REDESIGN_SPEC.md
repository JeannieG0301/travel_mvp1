# Web UI 改版需求（产品规格）

> 产品经理输出，供 Tech Lead 拆解任务、开发实现。
>
> 目标：改版后具备产品感，适合对外分享（如小红书等渠道）。
>
> **设计预览**：`frontend/preview/index.html`（静态可预览）；设计说明见 `UI_PREVIEW_NOTES.md`。
>
> **修订记录**：2026-02-22 配色回归规范（草地绿单主色、米白背景、去除蓝绿渐变），见 `UI_REVISION_NOTES.md`。

---

## 一、现状简要描述

当前 UI 偏工具感、缺乏旅行氛围：灰蓝配色、系统字体、扁平卡片，整体像内部表单工具。结果区按天展示虽清晰，但视觉层次弱，缺少品牌感和可分享的产品气质。

---

## 二、改版目标

希望达到：**有旅行感、专业可信、易上手**。用户打开页面能立刻感知「这是做新西兰行程的」，分享到小红书等渠道时具备可截图、可传播的产品形象。

---

## 三、具体改版点（按优先级）

| 优先级 | 改什么 | 改后效果 |
|--------|--------|----------|
| **P0** | **配色** | 从灰蓝改为更贴近新西兰/自然的色系。**主色**：草地绿 `#2d6a4f`（单主色，不使用蓝绿渐变）。**背景**：米白/暖灰 `#f8f6f3`。深色文字保持可读。去除蓝绿色系渐变与偏「科技蓝」的冷感。 |
| **P0** | **字体** | 标题使用有辨识度的字体（如思源宋体、霞鹜文楷或系统内类似衬线/手写感字体），正文保持清晰易读的无衬线。增强「旅行/人文」气质。 |
| **P0** | **头部/品牌区** | 增加简短 slogan 或副标题，强化「新西兰自驾」定位；适当加大标题字号、调整间距，形成清晰的品牌区。 |
| **P1** | **表单卡片** | 卡片增加轻微圆角、柔和阴影，与背景区分更明显；表单分组（必填 vs 选填）用视觉分隔更清晰，可选区保持 `details/summary` 但样式更柔和。 |
| **P1** | **按钮与输入控件** | 主按钮使用新主色，圆角与整体风格统一；输入框、下拉框边框颜色与主色呼应，hover 态有轻微反馈。 |
| **P1** | **结果区：行程卡片** | 按天行程卡片区分更明显：每天有标题栏（如「第 X 天」+ 当日主题），上午/下午/晚上用标签或小标题区分；交通/景点/活动/住宿可考虑分行或小图标（可选）增强可读性。 |
| **P2** | **结果区：总览与 tips** | 行程总览用引述或高亮样式突出；注意事项用列表或卡片式呈现，每条可读性更好。 |
| **P2** | **加载与错误态** | 加载动画与主色统一；错误提示样式与整体风格一致，不突兀。 |

---

## 四、参考与排除

**参考**：可借鉴旅行类产品（如马蜂窝、穷游、Klook）的配色和卡片层次感——暖色、自然感、信息分区清晰。不要求完全仿照。

**排除**：不做复杂动效（如页面过渡动画、微交互）；不引入重前端框架（保持当前 HTML/CSS/JS 结构）；不做大图背景或大量图片资源；不使用蓝绿渐变（优先用纯色配色与排版营造氛围）。

---

## 五、验收标准

1. **主色已替换**：页面主色从 `#2563eb` 改为指定的新西兰/自然系主色，整体无原蓝残留。
2. **字体已更新**：标题使用指定或替代字体，正文保持可读。
3. **表单分组清晰**：必填区与选填区视觉分隔明显，表单卡片与背景层次清晰。
4. **结果区可读性**：按天行程卡片有明确标题与时段区分，交通/景点/活动/住宿信息易扫读。
5. **响应式保持**：改版后移动端与桌面端布局正常，无错位或溢出。
